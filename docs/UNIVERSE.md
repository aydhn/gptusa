# USA Signal Bot - Universe

The `universe` module provides the foundational layer for managing which symbols (Stocks, ETFs) the USA Signal Bot will scan and generate signals for.

## Why a Local Universe?
At this phase, the bot relies on a local file-based universe. This approach ensures:
- **Offline & Reliable**: The bot can run scans without needing an external API to fetch thousands of ticker symbols.
- **Controlled Constraints**: Users can maintain a well-curated subset of highly liquid names, avoiding unpredictable micro-caps or unknown assets.
- **Foundation for Automation**: This phase builds the schema, validation, and filtering tools. Future phases can expand on this to automatically pull symbols from free providers (e.g., merging in Russell 3000 or S&P 500 lists) while applying the same strict filtering layer developed here.

## Watchlist vs. Full Universe
- **Watchlist (`watchlist.csv`)**: A small, user-maintained list of symbols that the user explicitly wants to track. This acts as the default if a broader universe is not built.
- **Full Universe (`default_universe.csv`)**: Generated by merging sample files and the watchlist using the `universe-build` command. It represents a broader snapshot of all available assets.

## Stock / ETF Separation
The `asset_type` column specifies whether an asset is a `stock` or `etf`. This allows for targeted scanning, e.g., only scanning ETFs for broad market signals.

## Active vs. Inactive
The `active` boolean flag allows a symbol to remain in the dataset for historical completeness but be easily filtered out of daily signal generation.

## CLI Commands
The universe can be managed via the CLI:
- `python -m usa_signal_bot universe-info`: View universe configuration and file locations.
- `python -m usa_signal_bot universe-validate`: Validate the default watchlist.
- `python -m usa_signal_bot universe-list`: List available active symbols.
- `python -m usa_signal_bot universe-build`: Build a consolidated universe snapshot from sample files.
- `python -m usa_signal_bot universe-summary`: Print an aggregated summary of the default universe.
