# Parameter Sensitivity

The Parameter Sensitivity module helps measure the impact of strategy parameter variations on backtest metrics.

## Goal
The primary goal is **not** to find the absolute "best" parameter, but rather to understand how the strategy behaves when parameters are tweaked. A strategy that is highly sensitive to tiny parameter changes is often overfitted and fragile, whereas a strategy with stable performance across a wide range of parameters is more robust.

## Non-Optimizer Approach
This system explicitly avoids any logic that automatically selects the "best parameters". The outputs generated by this module are stability maps, aggregate metrics, and lists of robust vs. fragile regions.

## Grid Types
- **SINGLE_PARAMETER:** Varies one parameter while keeping others at their default.
- **TWO_PARAMETER:** Varies two parameters simultaneously.
- **MULTI_PARAMETER:** Varies three or more parameters (use cautiously due to combinatorial explosion).
- **LOCAL_NEIGHBORHOOD:** Perturbs existing parameters around a base configuration by a specified radius.

## Metrics
- **Primary Metric:** The main metric to observe (e.g., `RETURN_PCT`, `WIN_RATE`).
- **Stability Score:** A metric derived from the variance and mean of the primary metric across completed grid cells. Higher scores indicate more stable parameters.

## CLI Usage
View sensitivity settings:
`python -m usa_signal_bot sensitivity-info`

Plan a grid without running:
`python -m usa_signal_bot parameter-grid-plan --strategy trend_following_rule --param min_score --values 45,50,55,60`

Run the grid (assuming candidate file exists):
`python -m usa_signal_bot sensitivity-run --strategy trend_following_rule --candidates-file data/candidates/selected.jsonl --param min_score --values 45,50,55,60 --write`
