# Confluence Engine

## Overview
The Confluence Engine aggregates multiple signals across different strategies to identify overlapping biases and multi-model consensus. The result is a `ConfluenceGroupResult` summarizing the collective strength and direction of the group.

## Aggregation Modes
Confluence can be evaluated across different boundaries:
- **BY_SYMBOL_TIMEFRAME (Default):** Aggregates all signals for a specific symbol on a specific timeframe.
- **BY_SYMBOL:** Aggregates signals across multiple timeframes for a single symbol.
- **BY_STRATEGY:** Aggregates all symbols/timeframes generated by a specific strategy.
- **GLOBAL:** Evaluates the aggregate momentum of the entire generated universe.

## Direction Bias
The Engine classifies the collective bias into one of several directions:
- `LONG_BIAS`: Overwhelming majority of long signals.
- `SHORT_BIAS`: Overwhelming majority of short signals.
- `WATCH_BIAS`: Signals are predominantly candidates to keep watching.
- `FLAT_BIAS`: Avoid or flatten current exposure.
- `CONFLICTED`: Simultaneous `LONG` and `SHORT` signals are present.
- `MIXED`: No clear single direction.

## Confluence Score and Strength
Confluence Score (0.0 to 100.0) is determined by the number of signals and their average confidence. The strength is then categorized into buckets:
- `NONE`
- `WEAK`
- `MODERATE`
- `STRONG`
- `VERY_STRONG`

> **Note:** The confluence engine identifies overlap; it does not issue trade orders. Conflict penalties are applied if opposing biases occur simultaneously.

## Example CLI Usage

Evaluate confluence of an existing signal file:
```bash
python -m usa_signal_bot signal-confluence --file data/signals/example.jsonl --mode by_symbol_timeframe --write
```

Run multiple strategies and evaluate their confluence directly:
```bash
python -m usa_signal_bot strategy-run-confluence --strategies trend_following_skeleton,momentum_skeleton --symbols AAPL,MSFT --timeframes 1d
```
