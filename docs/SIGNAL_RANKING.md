# Signal Ranking System

The Signal Ranking system provides a structured mechanism to score, penalize, and rank signals generated by rule-based strategies. It functions as a critical filter before a signal becomes a candidate for trading or research.

## Purpose

The primary goal of signal ranking is to translate a strategy's raw signal (with an initial score and confidence) into an actionable priority, adjusting for factors such as data quality, recency, confluence from multiple strategies, and risk profiles.

## Ranking Components

The rank score is bounded between `0.0` and `100.0` and is calculated using configurable weighted components:

1. **Signal Score**: The initial score assigned by the rule engine.
2. **Confidence**: The statistical or logical confidence behind the signal.
3. **Confluence**: An aggregation score from `signal_confluence` reflecting alignment among multiple indicators or strategies.
4. **Quality**: Evaluates missing data, volume anomalies, or structural issues.
5. **Recency**: A decay factor applied based on how old the signal is.
6. **Risk Penalty**: Reduces the score if specific risk flags (`HIGH_VOLATILITY`, `LOW_LIQUIDITY`, etc.) are detected.
7. **Action Priority**: Favors certain actions (e.g., `LONG` might be weighted higher than `WATCH`).

## Limits and Constraints

*   **No Backtesting Limit**: Because this system currently runs without strict historical backtesting verification, the rank score is artificially capped by `max_rank_score_without_backtest` (default `75.0`).
*   **Ranking is NOT Trade Execution**: A high-ranked signal implies it is mathematically interesting but it **does not represent a live order**.
*   **Quality Rejection**: A signal explicitly flagged as `REJECTED` by the quality guard is severely penalized and capped at a maximum score of `20.0`.

## CLI Usage Examples

Rank signals from a JSONL file and write the report to disk:
```bash
python -m usa_signal_bot signal-rank-file --file data/signals/example.jsonl --write
```
